802.11 management parameters

Parameters:
  • station_id - MAC ADDRESS
  • net_id - OCTET STRING, max 32
  • broadcast_addr - MAC BROADCAST ADDRESS
  • beacon_period - INTEGER, number of time units (1024 microseconds) between beacons, DEFAULT = 40.
  • number_of_peering - Unsigned32, indicates the number of mesh peering currently maintained by the STA,DEFAULT = 0.
  • accepting_additional_peerings - TruthValue,indicates whether the station is willing to accept additional peerings, DEFAULT = True.
  • max_retry - Unsigned32, specifies the maximum number of Peering Open retries that can be sent to establish a new peering instance, DEFAULT = 2.
  • retry_timeout - Unsigned32, specifies the initial retry timeout, in millisecond units, used by the Peering Open message, DEFAULT = 40.
  • confirm_timeout - Unsigned32, specifies the initial confirm timeout, in millisecond units, used waiting for a Peering Confirm message, DEFAULT = 40.
  • active_path_selection_protocol - INTEGER { hwmp (1), vendorSpecific (255) }, indicates the active path selection protocol, DEFAULT = 1.
  • active_path_selection_metric - INTEGER { airtimeLinkMetric (1), vendorSpecific (255) }, indicates the active path selection metric, DEFAULT = 1.
  • forwarding - TruthValue, specifies the ability of a mesh STA to forward MSDUs, DEFAULT = TRUE.
  • TTL - Unsigned32, specifies the value of TTL subfield set at a source STA.DEFAULT = 31.
  • active_congestion_control_mode - INTEGER {null(0), congestionControlSignaling (1), vendorSpecific (255) }, DEFAULT = 0.
  • active_syn_method - INTEGER { neighborOffsetSynchronization (1), vendorSpecific (255) }, DEFAULT = 1.

    TDMA parameters, They are only necessary for the MASTER

  • control_time - time between control frames
  • slots - number of slots
  • list_nodes - array of mac addresses of nodes, in the order of the slots each of them will use.
